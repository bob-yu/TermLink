Core Action Icon Options

These options only explore Add / Connect / Disconnect. They avoid plug-only shapes and try different interaction metaphors.

Set 1 - Session Window

Uses a terminal/session window as the base metaphor. Good when “serial port” is shown to users as a session tab.

Add
Connect
Disconnect
Most direct and professional. The downside is Add / Connect / Disconnect are action badges on the same terminal shape, so they can look similar at small size.

Set 2 - Channel Nodes

Represents connections as endpoints and channels. Better for local serial + remote serial + SSH/Telnet.

Add
Connect
Disconnect
Good abstraction for “connections” rather than physical plugs. Clean and modern. Connect/Disconnect are very readable.

Set 3 - Route Flow

Shows traffic route state. More dynamic and network-oriented.

Add
Connect
Disconnect
Looks modern, but may feel more like network routing than serial tooling. Best if remote access becomes the central product identity.

Set 4 - Control Symbols

Uses universal control metaphors. Very readable, less domain-specific.

Add
Connect
Disconnect
Simple and robust at small size. It is less expressive for serial/remote workflows, but it looks clean in a toolbar.

Set 5 - Light Plate

Small contained icons without heavy colored blocks. More polished than bare lines, less noisy than colored tiles.

Add
Connect
Disconnect
Best if you want the toolbar to feel more “designed” while staying restrained. Slightly more visual weight than pure monoline.

Set 6 - Status Dots

Uses session cards with status dots. Subtle, enterprise-tool style.
